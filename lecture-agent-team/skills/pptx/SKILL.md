## 1. 디자인 토큰

### 컬러 팔레트

| 토큰명 | HEX | 용도 |
|--------|-----|------|
| bg-light | #F5F0EB | 라이트 슬라이드 배경 (크림/베이지) |

| card-dark | #2A2A2A | 다크 모드 카드 배경 |
| card-light | #FFFFFF | 라이트 모드 카드 배경 |
| text-primary | #1A1A1A | 기본 텍스트 (라이트 모드) |
| text-inverse | #FFFFFF | 반전 텍스트 (다크 모드) |
| text-muted | #9CA3AF | 보조 텍스트 / 캡션 |
| accent-danger | #E53E3E | 경고, 문제점, 강조 (빨강) |
| accent-success | #48BB78 | 해결, 긍정, 연결됨 (초록) |

### 타이포그래피

| 요소 | 폰트 | 사이즈 | 굵기 |
|------|------|--------|------|
| 메인 타이틀 | Pretendard | 60–80pt | Black (900) |
| 섹션 헤딩 | Pretendard | 40–48pt | Bold (700) |
| 카드 타이틀 | Pretendard | 24–32pt | Bold (700) |
| 본문 텍스트 | Pretendard | 16–20pt | Regular (400) |
| 캡션/출처 | Pretendard | 12–14pt | Regular (400) |
| 뱃지 라벨 | Pretendard / Inter | 12–14pt | Medium (500), 영문 대문자 |
| 대형 숫자 | Pretendard / Inter | 80–120pt | Black (900) |

### 공통 스타일

- 카드 모서리: `border-radius 16px` (PptxGenJS: `rectRadius: 0.22`)
- 뱃지 모서리: pill shape (`rectRadius: 0.28`)
- 슬라이드 패딩: 좌우 0.6인치, 상하 0.5인치
- 카드 내부 패딩: 0.25–0.33인치
- 카드 간 간격: 0.17–0.25인치
- 그림자: 없음 (플랫 디자인)
- 테두리: 없음 (다크 카드는 선택적으로 `rgba(255,255,255,0.1)`)

## 2. 슬라이드 레이아웃 타입 (8종)

### TYPE-A: 타이틀 슬라이드

```
┌──────────────────────────────────────┐
│ [카테고리 뱃지] [WEEK N]     [로고] │
│                                      │
│  ██ 메인 타이틀 (2줄 이내)          │
│     초대형 Bold                      │
│                                      │
│  서브타이틀 설명문 (1–2줄)          │
└──────────────────────────────────────┘
배경: bg-light (#F5F0EB)
```

- 메인 타이틀: 60–80pt, Black
- 서브타이틀: 20pt, Regular, text-muted
- 좌측 상단 카테고리 + 회차 뱃지 / 우측 상단 로고(선택)
- 텍스트 좌측 정렬

### TYPE-B: 로드맵 / 목차 슬라이드

```
┌──────────────────────────────────────┐
│          [ROADMAP 뱃지]              │
│  ██ 섹션 타이틀                      │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐   │
│  │ 01  │ │ 02  │ │ 03  │ │ 04  │   │
│  │설명 │ │설명 │ │설명 │ │설명 │   │
│  └─────┘ └─────┘ └─────┘ └─────┘   │
└──────────────────────────────────────┘
배경: bg-dark (#1A1A1A)
```

- 카드 3–5개 가로 배열 / card-dark (#2A2A2A)
- 번호: 40pt Bold / 설명: 16pt Regular

### TYPE-C: 키 질문 / 인용 슬라이드

```
┌──────────────────────────────────────┐
│          [KEY QUESTION 뱃지]         │
│                                      │
│     "큰 인용문 텍스트                │
│      2줄 이내로 강렬하게"            │
│      ─────────────────               │
│      작은 캡션 텍스트                │
└──────────────────────────────────────┘
배경: bg-light (#F5F0EB)
```

- 중앙 정렬 / 인용문: 40–48pt Bold / 구분선: 폭 60%
- 캡션: 14pt, text-muted

### TYPE-D: 문제 제기 / 비교 카드

```
┌──────────────────────────────────────┐
│          [PROBLEM 뱃지]              │
│  ██ 섹션 타이틀                      │
│  ┌──────┐  ┌──────┐  ┌──────┐       │
│  │라벨A │  │라벨B │  │라벨C │       │
│  │상태 ✕│  │상태 ✕│  │상태 ✕│       │
│  └──────┘  └──────┘  └──────┘       │
│     ██ 하단 강조 메시지              │
└──────────────────────────────────────┘
배경: bg-light
```

- 카드 2–3개 / 상태: accent-danger + ✕
- 하단 강조: 32pt Bold, 중앙 정렬

### TYPE-E: 솔루션 / 연결 카드

```
┌──────────────────────────────────────┐
│          [SOLUTION 뱃지]             │
│  ██ 섹션 타이틀                      │
│  ┌──────┐🔗┌──────┐🔗┌──────┐       │
│  │연결 ✓│  │연결 ✓│  │연결 ✓│       │
│  └──────┘  └──────┘  └──────┘       │
│        ┌──────────────┐              │
│        │  핵심 키워드  │              │
│        └──────────────┘              │
└──────────────────────────────────────┘
배경: bg-dark (#1A1A1A)
```

- card-dark / 카드 간 🔗 연결 아이콘
- 상태: accent-success + ✓ / 하단 CTA 박스: card-light 배경

### TYPE-F: 정의 / 개념 설명 슬라이드

```
┌──────────────────────────────────────┐
│          [DEFINITION 뱃지]           │
│  ██ 섹션 타이틀                      │
│  ┃ "인용문 또는 정의 텍스트"         │
│  ┃ — 출처                            │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐   │
│  │키워드│ │키워드│ │키워드│ │키워드│   │
│  └─────┘ └─────┘ └─────┘ └─────┘   │
└──────────────────────────────────────┘
배경: bg-light
```

- 인용 블록: 좌측 수직선 3px, accent-brand / 이탤릭 20pt
- 하단 카드: 3–4개 / 키워드: 24pt Bold

### TYPE-G: 통계 / 성과 데이터 슬라이드

```
┌──────────────────────────────────────┐
│          [PERFORMANCE 뱃지]          │
│  ██ 섹션 타이틀                      │
│  ┌──────────┐ ┌──────────┐ ┌──────┐ │
│  │  90.2 %  │ │   90 %   │ │ 5-15x│ │
│  │ 보조설명 │ │ 보조설명 │ │설명  │ │
│  └──────────┘ └──────────┘ └──────┘ │
│                        출처 텍스트   │
└──────────────────────────────────────┘
배경: bg-dark (#1A1A1A)
```

- 숫자: 80–120pt Black, text-inverse / 단위: 32pt, text-muted
- 카드 2–3개 / 출처: 12pt 우측 하단

### TYPE-H: 다이어그램 / 시스템 구조

```
┌──────────────────────────────────────┐
│  ██ 섹션 타이틀                      │
│                                      │
│     [다이어그램 / 플로우차트 영역]   │
│     (이미지 삽입 또는 도형으로 구성) │
│                                      │

```

- 구조도, 비교도, 아키텍처 등
- 이미지 삽입 또는 PptxGenJS 도형으로 구성

## 4. 뱃지 시스템

- **위치**: 슬라이드 상단 중앙
- **모양**: pill shape
- **텍스트**: 영문 대문자만 (한국어 뱃지 금지)
- **다크 배경**: badge-bg-light (#E5E7EB) + 다크 텍스트
- **라이트 배경**: badge-bg (#374151) + 흰색 텍스트
- **패딩**: 좌우 24px, 상하 8px

| 뱃지 | 사용 상황 |
|------|----------|
| ROADMAP | 목차, 여정 |
| KEY QUESTION | 핵심 질문, 도입 |
| PROBLEM | 문제 제기 |
| SOLUTION | 해결책 |
| DEFINITION | 정의, 개념 |
| PERFORMANCE | 성과, 데이터 |
| CASE STUDY | 사례 |
| PRACTICE | 실습, 실전 |
| SUMMARY | 요약 |
| Q & A | 질의응답 |

## 5. 발표자 노트 (Presenter Notes)

모든 슬라이드에 `slide.addNotes()`로 발표자 노트를 입력한다.
노트 내용은 `lecture_script.md`의 해당 슬라이드 **발표 멘트** + **전환 멘트** 전문이다.

```javascript
slide.addNotes(
  "【발표 멘트】\n" +
  "...(발표 멘트 전문)...\n\n" +
  "【전환 멘트】\n" +
  "...(전환 멘트 전문)..."
);
```

- `【발표 멘트】` / `【전환 멘트】` 레이블로 구분
- 표지·참고문헌 슬라이드도 핵심 메시지 1문장 이상 입력

## 6. 금지 사항

- 한 슬라이드에 텍스트 7줄 이상 금지
- 밝은 배경 3장 연속 금지
- 카드 5개 초과 가로 배열 금지
- 그라데이션, 그림자 효과 금지 (플랫 디자인)
- 뱃지 없는 슬라이드 금지 (TYPE-A, TYPE-H 제외)
- 한국어 뱃지 금지
- 폰트 3종 이상 혼용 금지
